## 1.1 この教材の使い方

### 教材の構成と学習の進め方

この教材は、Linux を全く触ったことがない方を対象に、以下の 3 つの分野を基礎から順番に学べるように構成しています。

| 章 | テーマ | 学べること |
|---|---|---|
| 第2章 | シェル | Linux を操作するための「対話の仕組み」を理解する |
| 第3章 | コマンドリファレンス | 実際に使うコマンドを目的別に身につける |
| 第4章 | エディタ | ターミナル上でファイルを編集する技術を習得する |

最後の第5章には総合確認問題を用意しています。学んだ知識を組み合わせて、実践的な課題に取り組んでみましょう。

**おすすめの学習順序：**

1. まずこの「はじめに」を読んで、Linux の全体像をつかむ
2. 第2章「シェル」を読みながら、実際にターミナルを触ってみる
3. 第3章「コマンドリファレンス」で、よく使うコマンドを手を動かして覚える
4. 第4章「エディタ」で、ファイルの編集方法を身につける
5. 第5章「総合確認問題」で、理解度をチェックする

一度に全部を覚えようとする必要はありません。各章を行ったり来たりしながら、自分のペースで進めてください。

### 前提知識について

この教材を始めるにあたって、プログラミングやIT の専門知識は一切不要です。必要なのは以下の 2 つだけです。

- パソコン（Windows、Mac、どちらでもOK）を日常的に使えること
- キーボードで文字を入力できること

「コマンド」「ターミナル」「カーネル」...聞き慣れない言葉がたくさん出てきますが、すべてこの教材の中で説明します。分からない言葉が出てきても、焦らず読み進めてください。

### 学習環境の準備

この教材で学ぶ内容を実際に試すには、Linux が動く環境が必要です。環境の構築方法は「1.4 学習環境を整えよう」で詳しく説明しますので、まずはこの章を最後まで読んでから準備に取りかかりましょう。

## 1.2 Linuxとは何か

### OSの役割とLinuxの位置づけ

#### そもそも「OS」って何？

パソコンやスマートフォンを使っているとき、あなたは画面をクリックしたり、キーボードで文字を打ったりしていますよね。でも、その裏側では「ハードウェア」と呼ばれる機械の部品たち――CPU（計算する部品）、メモリ（一時的にデータを覚えておく部品）、ハードディスク（データを保存する部品）――が実際の仕事をしています。

しかし、人間がこれらの部品に直接命令を出すのは非常に難しいことです。そこで登場するのが **OS（オペレーティングシステム）** です。

**OS とは、「人間」と「コンピューターの部品（ハードウェア）」の間に立って、両者の通訳をしてくれるソフトウェア** のことです。

身近な例で考えてみましょう。

```
あなたの操作：「このファイルを保存したい」
    ↓
OS が翻訳：「ハードディスクのこの場所に、このデータを書き込んで」
    ↓
ハードウェアが実行：実際にデータを保存する
```

OS がなければ、ファイルを 1 つ保存するだけでも、ハードディスクのどのセクターに何バイト書き込むかを自分で指定しなければなりません。OS があるおかげで、私たちは「保存」ボタンを押すだけで済むのです。

#### Windows、Mac、Linux ― 3つのOSの違い

世の中には、主に 3 つの OS があります。

| OS | 開発元 | 主な使われ方 | 特徴 |
|---|---|---|---|
| **Windows** | Microsoft | 一般家庭・オフィスのPC | 最もシェアが大きい。市販ソフトが豊富 |
| **macOS** | Apple | Mac（Apple製PC） | デザイン・映像制作で人気。iPhoneとの連携が強い |
| **Linux** | コミュニティ（世界中の開発者） | サーバー・開発環境・組み込み機器 | 無料で使える。カスタマイズ自由。中身が公開されている |

Windows や macOS は、画面に表示されるアイコンやボタンを「マウスでクリック」して操作するのが基本です。この操作方法を **GUI（Graphical User Interface）** と呼びます。

一方、Linux はもちろん GUI でも操作できますが、**CUI（Character User Interface）** ――つまり、キーボードから文字（コマンド）を入力して操作する方法が中心です。この「文字で命令を出す画面」のことを **ターミナル** と呼びます。

> **ポイント：** Linux は「パソコンに入っている OS」というより、「世界中のサーバー（インターネットを支えるコンピューター）の大半で動いている OS」と考えると、その重要性が分かりやすいでしょう。

#### Linux を一言でいうと

**Linux とは、世界中の開発者が協力して作り上げている、無料で誰でも使える OS** です。

- **無料：** お金を払わずにダウンロードして使えます
- **オープンソース：** OS の設計図（ソースコード）が公開されていて、誰でも中身を見たり改良したりできます
- **高い信頼性：** 24 時間 365 日止まってはいけないサーバーで使われるほど安定しています

### Linuxの歴史

Linux の歴史を知っておくと、「なぜ Linux はこういう仕組みなのか」が理解しやすくなります。ざっくりとした流れを見てみましょう。

**1969年 ― UNIXの誕生**

アメリカの AT&T ベル研究所で、**UNIX（ユニックス）** という OS が誕生しました。UNIX は「小さな道具（プログラム）を組み合わせて大きな仕事をする」という設計思想を持っていました。この考え方は、現在の Linux にもそのまま受け継がれています。

**1983年 ― GNUプロジェクトの開始**

リチャード・ストールマンという人物が、「誰もが自由に使えるソフトウェアを作ろう」という理念のもと、**GNU（グヌー）プロジェクト** を立ち上げました。ここで、OS を構成するさまざまなツール（コマンドやコンパイラなど）が作られました。

**1991年 ― Linuxカーネルの誕生**

フィンランドの大学生だった **リーナス・トーバルズ（Linus Torvalds）** が、OS の心臓部である「カーネル」を開発しました。これが **Linux カーネル** です。

GNU プロジェクトのツールと Linux カーネルが組み合わさることで、完全に自由に使える OS ―― 今日の「Linux」が誕生しました。

```
GNU のツール（コマンド、ライブラリなど）
        ＋
Linux カーネル（OS の心臓部）
        ＝
今日の「Linux」（正式には GNU/Linux）
```

### Linuxが使われている場所

「Linux なんて聞いたことがない」という方も、実は毎日 Linux のお世話になっています。

- **Webサーバー：** Google、Amazon、YouTube...あなたが見ているほとんどの Web サイトは、Linux で動いているサーバーから配信されています
- **スマートフォン：** Android は Linux カーネルをベースに作られています。つまり、Android スマホは Linux で動いているのです
- **スーパーコンピューター：** 世界のスーパーコンピューターの上位 500 台のほぼ 100% が Linux を使っています
- **組み込み機器：** テレビ、カーナビ、ルーターなど、身の回りの電子機器の多くに Linux が入っています
- **クラウドサービス：** AWS、Google Cloud、Azure といったクラウドの基盤の大部分が Linux です

> **ポイント：** Linux を学ぶということは、インターネットを支えるインフラ技術の基礎を学ぶということです。

## 1.3 Linuxディストリビューション

### ディストリビューションとは

Linux カーネルは OS の「心臓部」に過ぎません。実際にパソコンとして使うには、カーネルの他にもたくさんのソフトウェア（コマンド、エディタ、パッケージ管理ツールなど）が必要です。

これらを一式まとめて、「インストールしたらすぐ使える状態」にしたパッケージのことを **ディストリビューション（distribution = 配布物）** と呼びます。

料理に例えると：

```
Linux カーネル ＝ お米（主食・基本）
ディストリビューション ＝ お弁当（お米＋おかず＋箸がセットになったもの）
```

同じお米（Linux カーネル）を使っていても、お弁当の中身（付属するソフトウェアや設定）が違えば、別のディストリビューションになります。

### 代表的なディストリビューション

数百以上のディストリビューションが存在しますが、初心者が知っておくべき代表的なものを紹介します。

| ディストリビューション | 特徴 | こんな人向け |
|---|---|---|
| **Ubuntu（ウブントゥ）** | 初心者に最も優しい。情報が豊富。デスクトップ用途にも強い | Linuxを初めて触る人 |
| **Debian（デビアン）** | Ubuntu の元になったディストリビューション。安定性重視 | 安定した環境を求める人 |
| **CentOS / Rocky Linux / AlmaLinux** | 企業のサーバーでよく使われる Red Hat 系 | サーバー構築を学びたい人 |
| **Fedora（フェドラ）** | 最新技術をいち早く取り入れる。Red Hat の実験的な位置づけ | 新しい技術を試したい人 |
| **Arch Linux** | 自分で一から構築する上級者向け | 仕組みを深く理解したい人 |

### 初心者におすすめの選び方

迷ったら **Ubuntu** を選んでください。理由は次の通りです。

1. **日本語の情報が圧倒的に多い** ― 困ったときに検索すれば、ほぼ確実に日本語の解決策が見つかります
2. **インストールが簡単** ― 画面の指示に従うだけでセットアップできます
3. **コミュニティが大きい** ― 質問できる場所（フォーラム、Q&A サイト）が充実しています
4. **デスクトップ環境が整っている** ― GUI も充実しているので、最初のハードルが低いです

> **ポイント：** この教材では、特定のディストリビューションに依存しない「共通の基礎知識」を中心に解説します。どのディストリビューションを選んでも、ここで学ぶ内容はそのまま使えます。

## 1.4 学習環境を整えよう

Linux を学ぶには、実際に触れる環境が必要です。いくつかの方法がありますので、自分の状況に合ったものを選んでください。

### 方法1：仮想マシンを使う（おすすめ）

**仮想マシン** とは、今使っているパソコンの中に「もう1台のパソコン」を作る技術です。仮想マシンの中に Linux をインストールすれば、今の環境（Windows や Mac）はそのままで Linux を試せます。

**メリット：**
- 今のパソコンの環境を壊す心配がない
- 失敗しても仮想マシンを削除してやり直せる
- 実際の Linux と同じ体験ができる

**使うソフトウェア：**

| ソフトウェア | 対応OS | 特徴 |
|---|---|---|
| **VirtualBox** | Windows / Mac（Intel） | 無料で高機能。情報も豊富 |
| **UTM** | Mac（Apple Silicon / Intel） | M1/M2/M3 Mac に最適。無料 |

**ざっくりした手順：**
1. VirtualBox（または UTM）をインストール
2. Ubuntu のインストール用イメージ（ISO ファイル）をダウンロード
3. VirtualBox で新しい仮想マシンを作成し、Ubuntu をインストール
4. ターミナルを起動して、Linux の世界へ！

### 方法2：WSL2を使う（Windowsユーザー向け）

Windows 10/11 を使っている方は、**WSL2（Windows Subsystem for Linux 2）** という機能を使えば、Windows の中で直接 Linux を動かすことができます。

**メリット：**
- Windows と Linux を同時に使える
- インストールが非常に簡単
- 動作が軽い

**ざっくりした手順：**
1. PowerShell を管理者として開く
2. `wsl --install` コマンドを実行
3. パソコンを再起動
4. Ubuntu が自動的にインストールされ、すぐに使い始められる

### 方法3：クラウド環境を使う

パソコンのスペックが低い場合や、手軽に試したい場合は、クラウド上の Linux サーバーを使う方法もあります。

- **AWS（Amazon Web Services）** ― 無料利用枠あり
- **Google Cloud Platform** ― 無料トライアルあり
- **さくらのVPS** ― 月額数百円から

ただし、クラウド環境はネットワークの知識も少し必要になるため、最初は方法1か方法2をおすすめします。

### ターミナルエミュレータの基本操作

環境を用意したら、最初に触れるのが **ターミナルエミュレータ**（通称：ターミナル）です。

ターミナルとは、コマンドを入力して Linux を操作するためのアプリケーションです。見た目は「黒い背景に白い文字」のシンプルな画面です。

```
user@ubuntu:~$ _
```

この表示を **プロンプト** と呼びます。「準備ができたよ、命令をどうぞ」という意味です。

- `user` ― 今ログインしているユーザー名
- `ubuntu` ― コンピューターの名前（ホスト名）
- `~` ― 今いる場所（ホームディレクトリ）
- `$` ― 一般ユーザーであることを示す記号（管理者の場合は `#`）

ここにコマンドを入力して Enter キーを押すと、Linux がそのコマンドを実行してくれます。

## なぜエンジニアは「黒い画面」を使うのか？

Linux の学習を始めると、多くの初心者が抱く疑問があります。

> 「なぜ、わざわざマウスを使わずに、文字だけの黒い画面（ターミナル）で操作するの？ 不便じゃないの？」

とても自然な疑問です。でも、実はターミナルには GUI（マウス操作）にはない、大きなメリットがあるのです。

### 理由1：圧倒的に速い

例えば、「あるフォルダの中にある画像ファイル 500 個の名前を、すべて連番に変更したい」としましょう。

**GUI（マウス操作）の場合：**
1. ファイルを1つ右クリック → 「名前の変更」を選択 → 新しい名前を入力
2. これを 500 回繰り返す...

**ターミナル（コマンド）の場合：**
```bash
n=1; for f in *.jpg; do mv "$f" "image_$(printf '%03d' $n).jpg"; n=$((n+1)); done
```
たった1行のコマンドで、500 個のファイル名が一瞬で変更されます。

### 理由2：正確に伝えられる・再現できる

GUI の操作を人に伝えるとき、「ここのメニューを開いて、このボタンを押して...」とスクリーンショット付きで長い手順書を書く必要があります。しかも、ソフトのバージョンが変わると画面が変わってしまうことも。

一方、コマンドなら：
```bash
sudo apt update && sudo apt upgrade -y
```
この 1 行をコピーして渡すだけで、誰が実行しても同じ結果になります。

### 理由3：自動化できる

コマンドは「ただの文字列」です。文字列は、ファイルに書き出してまとめて実行することができます。これが **シェルスクリプト** です。

- 「毎日深夜3時にデータのバックアップを取る」
- 「新しいメンバーが入ったら、アカウントを作成してフォルダを用意する」
- 「サーバーに異常があったらメールで通知する」

こうした作業を自動化できるのは、コマンドベースで操作しているからこそです。

### 理由4：リモートで操作できる

サーバーは、目の前にある自分のパソコンとは限りません。世界中のデータセンターに置かれていて、ネットワーク越しに操作します。

このとき使うのが **SSH** という仕組みです。SSH で接続すると、手元のターミナルに文字を打ち込むだけで、遠く離れたサーバーを操作できます。GUI を転送するには大量のデータ通信が必要ですが、文字だけなら通信量はごくわずかです。

### 理由5：「何をしたか」が記録に残る

ターミナルで実行したコマンドは、すべて履歴として記録されます。「3日前にどんな操作をしたか」「どのコマンドでこの設定を変更したか」を後から振り返ることができます。

---

> まとめると、**ターミナル（黒い画面）は「不便な古いやり方」ではなく、「速く・正確に・自動で・どこからでも」コンピューターを操作するための、プロの道具** なのです。

## 学習を始める前に ― 大切なマインドセット

最後に、これから Linux の学習を始めるあなたに、いくつかの大切なことを伝えさせてください。

### 最初は分からなくて当たり前です

Linux に初めて触れると、見慣れない英単語、意味不明な記号、真っ黒な画面...何もかもが「???」だと思います。

**それで大丈夫です。**

ベテランのエンジニアだって、最初はみんな同じところからスタートしています。今は「何が分からないかも分からない」状態かもしれませんが、1 つずつ学んでいくうちに、少しずつ全体像が見えてきます。

### 「覚える」より「慣れる」

コマンドを一覧表にして暗記しようとする必要はありません。よく使うコマンドは、使っているうちに自然と指が覚えます。最初のうちは「あのコマンド、なんだっけ？」と調べながらで全く問題ありません。プロのエンジニアでも、毎日のように `man`（マニュアル）コマンドを使ったり、ネットで検索したりしています。

### 実際に手を動かそう

この教材を「読むだけ」で終わらせないでください。書いてあるコマンドを、自分のターミナルに打ち込んで、結果を確認してください。うまくいかなかったら、なぜうまくいかないのかを考えてみてください。その「試行錯誤」こそが、最も効果的な学習方法です。

### エラーは敵ではなく、先生です

コマンドを間違えると、英語でエラーメッセージが表示されます。最初は怖く感じるかもしれませんが、エラーメッセージは「何が問題なのか」を教えてくれる貴重な情報です。

```
bash: cta: command not found
```

たとえばこのエラーは、「`cta` というコマンドは見つかりませんよ」と言っています。`cat` を `cta` と打ち間違えただけですね。エラーを恐れず、メッセージをよく読む習慣をつけましょう。

### 完璧を目指さなくていい

「全部理解してから次に進もう」と思うと、途中で挫折しやすくなります。70% くらい理解できたら、次に進んでしまって大丈夫です。後の章で学ぶ内容が、前の章の理解を助けてくれることもよくあります。
